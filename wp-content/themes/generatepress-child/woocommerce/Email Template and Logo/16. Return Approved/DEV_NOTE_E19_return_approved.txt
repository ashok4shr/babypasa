E19 Return Approved

TRIGGER
-------
Type   : Manual admin action
Source : Admin reviews return request (stored from E18) and
         approves it via WooCommerce order panel or custom button

  // Add "Approve Return" button to WooCommerce order panel
  add_action( 'woocommerce_order_actions',
  function( $actions ) {
      $actions['babypasa_approve_return'] = 'Approve Return Request';
      return $actions;
  });

  add_action( 'woocommerce_order_action_babypasa_approve_return',
  function( $order ) {
      $order_id = $order->get_id();

      // Guard: return must have been requested first
      if ( ! get_post_meta( $order_id, '_return_requested', true ) ) return;
      // Guard: don't approve twice
      if ( get_post_meta( $order_id, '_return_approved', true ) ) return;

      // Mark as approved
      update_post_meta( $order_id, '_return_approved', '1' );
      update_post_meta( $order_id, '_return_approved_at', time() );

      babypasa_send_e19( $order_id, $order );
  });

====================================================================
SEND FUNCTION
====================================================================

  function babypasa_send_e19( $order_id, $order = null ) {
      if ( get_post_meta( $order_id, '_e19_sent', true ) ) return;
      if ( ! $order ) $order = wc_get_order( $order_id );
      if ( ! $order ) return;

      $first_name   = $order->get_billing_first_name();
      $order_number = $order->get_order_number();

      // Read return items stored in E18
      $return_items_raw = get_post_meta(
          $order_id, '_return_items', true );
      $return_items = $return_items_raw
          ? json_decode( $return_items_raw, true )
          : [];

      // Fallback: if no specific return items stored,
      // use all order items
      if ( empty( $return_items ) ) {
          foreach ( $order->get_items() as $item ) {
              $return_items[] = [
                  'name' => $item->get_name(),
                  'qty'  => $item->get_quantity(),
              ];
          }
      }

      // Upaya branch locator URL
      // DEVELOPER: Confirm correct URL with Upaya City Cargo
      // Use their official branch finder page if available
      // Fallback: Upaya contact page
      $branch_url  = 'https://upayacargo.com/branches';
      $pickup_url  = 'mailto:support@babypasa.com'
                     . '?subject=Return Pickup Request — Order #'
                     . $order_number;
      $support_url = 'mailto:support@babypasa.com';

      babypasa_send_email(
          $order->get_billing_email(),
          'Your return request has been approved — Order #'
              . $order_number . ' ✅',
          'e19-return-approved',
          compact(
              'first_name', 'order_number',
              'return_items', 'branch_url',
              'pickup_url', 'support_url'
          )
      );

      update_post_meta( $order_id, '_e19_sent', '1' );
  }

====================================================================
TWO RETURN OPTIONS — HOW EACH FLOWS
====================================================================

OPTION 1 — PICKUP BY UPAYA
CTA: "Request Pickup" → mailto:support@babypasa.com
     Subject pre-filled: "Return Pickup Request — Order #XXXXX"

Flow:
  1. Customer clicks → email opens to support@babypasa.com
  2. Support receives email → contacts Upaya to arrange collection
  3. Upaya visits customer address, collects parcel
  4. Upaya scans parcel in → webhook fires
  5. E20 fires when parcel arrives at warehouse

Admin effort: One email to Upaya per pickup request.
No automated flow needed.

OPTION 2 — DROP OFF AT UPAYA BRANCH
CTA: "Find a Branch" → Upaya branch locator URL

Flow:
  1. Customer visits nearest Upaya branch, hands over parcel
  2. Upaya scans parcel in → webhook fires automatically
  3. E20 fires when parcel arrives at warehouse

Admin effort: Zero. Fully self-service.

IMPORTANT — YOU DON'T NEED TO KNOW WHICH OPTION THEY CHOOSE
In both cases, the parcel eventually arrives at the warehouse
and triggers E20 via Upaya webhook. The return flow is identical
regardless of collection method. No admin action needed unless
the customer explicitly requests a pickup.
