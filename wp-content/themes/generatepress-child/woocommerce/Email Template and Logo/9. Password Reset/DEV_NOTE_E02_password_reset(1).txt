BABYPASA — DEVELOPER NOTE
E02 Password Reset
====================================================================

TRIGGER
-------
Type   : WooCommerce native hook
Hook   : woocommerce_reset_password_notification
Source : Fired when customer clicks "Lost your password?" on
         /my-account/lost-password/ and submits their email

OVERRIDE WOOCOMMERCE DEFAULT
-----------------------------
WooCommerce sends its own password reset email by default.
To use the BabyPasa template instead, disable the default and
hook in your custom send:

  // Disable WooCommerce default password reset email
  add_filter( 'woocommerce_email_classes', function( $emails ) {
      unset( $emails['WC_Email_Customer_Reset_Password'] );
      return $emails;
  });

  // Send BabyPasa E02 template instead
  add_action( 'woocommerce_reset_password_notification',
  function( $user_login, $reset_key ) {
      $user = get_user_by( 'login', $user_login );
      if ( ! $user ) return;

      $first_name        = get_user_meta( $user->ID, 'first_name', true )
                           ?: $user->display_name;
      $reset_link        = network_site_url(
          'wp-login.php?action=rp&key=' . $reset_key
          . '&login=' . rawurlencode( $user_login ), 'login'
      );

      // Or use WooCommerce account URL (cleaner for shop context):
      $reset_link = add_query_arg([
          'key'   => $reset_key,
          'login' => rawurlencode( $user_login ),
      ], wc_get_endpoint_url(
          'lost-password', '', wc_get_page_permalink('myaccount')
      ));

      babypasa_send_email(
          $user->user_email,
          'Reset your BabyPasa.Com password',
          'e02-password-reset',
          compact( 'first_name', 'reset_link' )
      );
  }, 10, 2 );

LINK EXPIRY
-----------
WooCommerce password reset keys expire after 24 hours by default.
This is controlled by the WordPress core function
get_password_reset_key() — no custom code needed.

The 24-hour expiry is already stated in the email body. No changes
required to WordPress/WooCommerce defaults.

SECURITY NOTES
--------------
1. Never log or store the reset link — it is a one-time token.
2. The "Didn't request this?" banner is intentionally calm and pink
   (not red/amber) — avoid alarming customers who simply mistyped
   their email on the lost password form.
3. The raw URL fallback below the button is important — some email
   clients (particularly older corporate Outlook) block button links.
   Without the fallback, those users cannot reset their password.
4. Do NOT include the reset key in the subject line.

WHAT THIS EMAIL DOES NOT HAVE
------------------------------
- No journey tracker
- No order items
- No payment tiles
- No product information
- Short by design — customer just wants the link

SUBJECT LINE
------------
Reset your BabyPasa.Com password

(No emoji, no urgency — subject line urgency reads as phishing.
Expiry is stated clearly inside the email body instead.)

PLACEHOLDERS USED
-----------------
{{first_name}}          get_user_meta( $user->ID, 'first_name', true )
{{password_reset_link}} Generated reset URL (see code above)

NEXT EMAIL IN SEQUENCE
-----------------------
None — E02 does not trigger any follow-up email.
